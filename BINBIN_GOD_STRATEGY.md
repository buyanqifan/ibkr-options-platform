# Binbin God Strategy v0.1.0 - 使用文档

## 📖 策略介绍

**彬彬神策略**（Binbin God Strategy）是一个智能化的 Wheel 策略增强版本，专门从 **MAG7** 股票池中动态选择最优标的进行 Wheel 操作。

### MAG7 股票池

- **MSFT** - Microsoft（微软）
- **AAPL** - Apple（苹果）
- **NVDA** - NVIDIA（英伟达）
- **GOOGL** - Alphabet/Google（谷歌）
- **AMZN** - Amazon（亚马逊）
- **META** - Meta/Facebook（脸书）
- **TSLA** - Tesla（特斯拉）

## 🎯 核心特性

### 1. 智能选股系统

基于四大维度量化评分：

```
┌─────────────────────────────────────┐
│ 选股评分体系                         │
├─────────────────────────────────────┤
│ P/E Ratio (20%)   价值因子          │
│ Option IV (40%)   权利金收入        │
│ Momentum (20%)    趋势方向          │
│ Stability (20%)   风险管理          │
└─────────────────────────────────────┘
```

#### 评分逻辑

**P/E 评分（越低越好）**
- PE < 20: 100 分
- PE > 70: 20 分
- 线性插值

**IV 评分（越高越好）**
- IV Rank × 60% + IV Percentile × 40%
- 高 IV = 高额权利金收入

**动量评分（正向偏好）**
- 1 个月涨幅 × 60% + 3 个月涨幅 × 40%
- 适度正增长最佳（避免过热）

**稳定性评分（低风险偏好）**
- Beta × 60% + 股息率 × 40%
- 低 Beta + 高股息 = 稳定

---

### 2. 动态调仓机制

**何时调仓？**

当当前持仓股票的评分比最优股票评分低 **15%** 以上时，自动切换到更优标的。

```python
if (best_stock_score - current_stock_score) > 15%:
    switch_to(best_stock)
```

**可调参数：**
- `rebalance_threshold`: 默认 15%，可在 UI 中配置

---

### 3. Wheel 策略核心

完整的 Wheel 循环：

```
Sell Put → 被行权 → 买入股票 → Sell Call → 被行权 → 卖出股票 → Sell Put → ...
```

**SP 阶段（Sell Put）**
- 卖出虚值 Put，收取权利金
- 目标：被行权获得股票，或权利金到期

**CC 阶段（Covered Call）**
- 卖出虚值 Call，收取权利金
- 目标：被行权卖出股票，或权利金到期

**CC 优化功能**（新增）
- 当股票价格低于成本基础时自动优化策略
- 动态调整 delta 范围：从 0.30 降低到 0.15，寻求更高行权价格
- 设置最小行权价格约束，尽量让行权价格 ≥ 成本基础的 98%
- 确保在被行权时减少损失，同时保持权利金收入

**配置参数**：
- `cc_optimization_enabled`: 启用 CC 优化（默认 true）
- `cc_min_delta_cost`: 最小 delta 值（默认 0.15）
- `cc_cost_basis_threshold`: 触发优化的价格折扣阈值（默认 0.05，即 5%折扣）
- `cc_min_strike_premium`: 最小权利金比例（默认 0.02，即 2%）

## ⚙️ 配置参数说明

### 基础参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| Initial Capital | $150,000 | 初始资金 |
| Max Leverage | 1.0 | 最大杠杆倍数 |
| Date Range | 2025-01-01 ~ 2026-03-05 | 回测时间范围 |

### Wheel 参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| DTE Min | 30 天 | 最短到期天数 |
| DTE Max | 45 天 | 最长到期天数 |
| Put Delta | 0.30 | 卖出 Put 的目标 Delta |
| Call Delta | 0.30 | 卖出 Call 的目标 Delta |
| Max Positions | 10 | 最大同时持仓数 |

### 特色参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| **Rebalance Threshold** | **15%** | **调仓阈值（核心参数）** |

**调仓阈值详解：**

- **15%** 表示：只有当新股票评分比当前股票高 **15% 以上**时才调仓
- **降低此值**（如 10%）：更频繁调仓，捕捉更多机会
- **提高此值**（如 25%）：减少调仓，降低交易成本
- **设置为 0%**：始终持有评分最高的股票

### 评分公式

```python
# P/E 评分（逆函数）
pe_score = 100 - (PE - 10) × (80 / 70)
clamped to [20, 100]

# IV 评分（加权平均）
iv_score = 0.6 × IV_Rank + 0.4 × IV_Percentile

# 动量评分（分段线性）
momentum = 0.6 × change_1m + 0.4 × change_3m
if momentum <= -10%: score = 20
elif momentum >= +10%: score = 80
else: score = 50 + momentum × 3

# 稳定性评分（Beta + 股息）
beta_score = f(Beta)  # 低 Beta 高分
div_score = min(100, dividend_yield × 20)
stability_score = 0.6 × beta_score + 0.4 × div_score

# 总分（加权平均）
total_score = 0.2×pe + 0.4×iv + 0.2×momentum + 0.2×stability
```

## 📈 性能优化建议

### 1. 资金规模

**推荐资金：** $100,000 - $500,000

**原因：**
- MAG7 股票价格较高（$200-$800）
- 每手 100 股，需要足够保证金
- 分散多只股票需要充足资金

---

### 2. 回测周期

**推荐周期：** 6 个月 - 2 年

**原因：**
- 太短（<3 个月）：无法体现策略优势
- 太长（>3 年）：MAG7 基本面可能变化
- 默认 14 个月（2025-01 到 2026-03）是合理测试期

## ⚠️ 风险提示

### 1. 模型风险

- **历史数据不代表未来表现**
- **合成数据仅用于测试，不可用于实盘**
- **评分模型基于简化假设**

### 2. 市场风险

- **MAG7 集中度高** - 全部为科技巨头，相关性高
- **系统性风险** - 科技股整体下跌时无法规避
- **流动性风险** - 极端行情下期权流动性枯竭

### 3. 操作风险

- **频繁调仓增加交易成本**
- **过度拟合历史数据**
- **参数设置不当导致亏损**
