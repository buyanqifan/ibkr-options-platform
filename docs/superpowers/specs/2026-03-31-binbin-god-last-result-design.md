# Binbin God 最近一次回测结果服务端持久化设计

## 概要

为 `binbin_god` 页面增加一套轻量的服务端持久化能力，用于保存“最近一次成功回测结果”。目标是让用户在刷新页面、关闭浏览器后重新打开页面时，仍然能看到上一轮回测的结果和对应参数，而不依赖浏览器本地存储，也不把每次运行都写入通用回测历史表。

这套能力只服务 `binbin_god` 页面，不扩展到其他回测页面，不改变现有 `BacktestStorage` 历史记录体系。

## 设计目标

- 刷新页面后恢复上一轮成功回测结果
- 关闭浏览器并重新打开后仍可恢复
- 恢复结果时，同时恢复参数面板输入值
- 不污染通用 `backtest_history`
- 存储实现尽量轻量，可维护成本低

## 不做的事

- 不给普通 `backtester` 页面增加同样能力
- 不把“最近一次结果”写入 `BacktestStorage`
- 不做多用户隔离或账号维度隔离
- 不提供历史版本管理，只保留最近一条

## 方案概述

新增一个轻量服务端存储模块，例如：

- `core/backtesting/last_result_store.py`

该模块只负责读写一份 JSON 文件，保存 `binbin_god` 最近一次成功回测的完整 payload。文件内容包含：

- `saved_at`
- `params`
- `result`

当页面运行回测成功时，自动覆盖保存这份文件。页面加载时，自动读取文件，如果存在且格式有效，就恢复结果区、导出按钮状态、MAG7 分析区，以及参数面板的当前值。

## 文件与职责

### 1. 轻量持久化模块

新增：

- `core/backtesting/last_result_store.py`

提供以下接口：

- `save_last_binbin_god_result(params: dict, result: dict) -> None`
- `load_last_binbin_god_result() -> dict | None`
- 可选：`clear_last_binbin_god_result() -> None`

实现要求：

- 使用项目本地 JSON 文件持久化
- 文件路径固定且集中定义
- 写入时确保 JSON 可序列化
- 文件不存在时返回 `None`
- 文件损坏或读取失败时记录日志并返回 `None`

### 2. Binbin God 页面恢复逻辑

修改：

- `app/pages/binbin_god.py`

改动点：

- 现有回测成功回调在生成结果 UI 后，额外调用 `save_last_binbin_god_result()`
- 页面增加一个“首次加载恢复最近结果”的回调
- 恢复时：
  - 更新 `binbin-results-store`
  - 更新 `binbin-params-store`
  - 重建结果区 children
  - 显示导出按钮
  - 恢复表单输入值

恢复逻辑必须复用现有结果渲染路径，避免页面有两套不同的结果构造方式。

## 恢复行为

页面首次加载时：

- 如果没有最近结果文件：保持当前默认空页面
- 如果有最近结果文件且结构有效：
  - 自动显示上一轮结果
  - 参数输入框恢复为对应参数
  - 用户无需再次点击 Run

页面再次运行新回测时：

- 用最新成功结果覆盖旧文件

回测失败时：

- 不覆盖已有最近结果
- 页面按现有错误提示展示

## 数据格式

JSON 文件结构固定如下：

```json
{
  "saved_at": "2026-03-31T12:34:56.000000",
  "params": { "...": "..." },
  "result": { "...": "..." }
}
```

要求：

- `params` 直接保存页面提交给引擎的参数
- `result` 直接保存页面当前依赖的结果结构
- 不额外做复杂归一化

## UI 与交互约束

- 不增加新的显式“恢复”按钮
- 不增加新的页面模块或导航项
- 不改变当前页面视觉布局
- 默认自动恢复最近一次成功结果

## 错误处理

如果最近结果文件出现以下情况：

- 文件不存在
- JSON 解析失败
- 缺少 `params` 或 `result`

则页面应：

- 静默回退到默认空状态
- 不抛出前端错误
- 不阻止用户继续运行新回测

存储模块应记录日志，便于后续排查。

## 测试方案

新增测试覆盖两层：

### 1. 持久化模块测试

新增例如：

- `tests/unit/test_last_result_store.py`

覆盖：

- 成功保存并读取最近结果
- 文件不存在时返回 `None`
- 文件损坏时返回 `None`

### 2. 页面测试

补充到：

- `tests/unit/test_binbin_god_page.py`

覆盖：

- 回测成功后会调用最近结果保存
- 页面首次加载时会恢复最近结果
- 恢复后结果区不为空
- 恢复后导出按钮显示
- 恢复后参数值按最近一次结果回填

## 假设与决策

- 当前系统默认单用户使用场景，最近结果无需区分用户
- 最近结果只保留一条，不保留历史版本
- 最近结果存储是服务端文件，不使用浏览器 localStorage/sessionStorage
- 当前页面已有的 `dcc.Store` 继续保留，但只作为当前会话内状态载体；真正的跨刷新恢复依赖服务端文件
